Components, Props & Render

What are Components?

Components are reusable pieces of UI. Think of them like LEGO blocks - you can combine them to build complex interfaces.

A Simple Component
// This is a component - a function that returns UI
function Welcome() {
  return (
    <div>
      <h1>Hello, World!</h1>
      <p>Welcome to React</p>
    </div>
  );
}

What are Props?

Props (short for "properties") are how you pass data into components. They're like arguments to a function.

Without Props (Static)
function Greeting() {
  return <h1>Hello, John!</h1>;
}
With Props (Dynamic)
function Greeting(props) {
  return <h1>Hello, {props.name}!</h1>;
}

// Usage:
<Greeting name="John" />
<Greeting name="Sarah" />

Real Example: User Card 

function UserCard(props) {
  return (
    <div className="card">
      <img src={props.avatar} />
      <h2>{props.name}</h2>
      <p>{props.bio}</p>
    </div>
  );
}

// Use it multiple times with different data:
<UserCard 
  name="Alice" 
  avatar="alice.jpg" 
  bio="Developer" 
/>
<UserCard 
  name="Bob" 
  avatar="bob.jpg" 
  bio="Designer" 
/>

Key Concepts:

  • Components are functions that return JSX (HTML-like code)
  • Props make components reusable with different data
  • When props change, React automatically re-renders the component
  • Props are read-only - you can't change them inside the component

Strict Mode

What is Strict Mode?

Strict Mode is a development tool that helps you find problems in your code. It's like having a helpful assistant that points out potential bugs.

Important:

Strict Mode only works during development. When you deploy your app to production, it automatically turns off.

How to Enable Strict Mode
import React from 'react';

function App() {
  return (
    <React.StrictMode>
      <YourApp />
    </React.StrictMode>
  );
}

What Does Strict Mode Do?

  • Identifies unsafe code - Warns you about outdated patterns
  • Detects unexpected side effects - Helps catch bugs early
  • Warns about deprecated features - Keeps your code modern
  • Runs components twice - Tests that they work correctly

Why Run Components Twice?

React runs your components twice in development to make sure they're "pure" - meaning they don't have unexpected side effects.

❌ Bad (Will Cause Issues)

let count = 0;

function Counter() {
  count++; // Don't do this!
  return <div>{count}</div>;
}

This will give unexpected results because the variable changes every render.

✅ Good (Works Correctly)

function Counter() {
  const [count, setCount] = useState(0);
  
  return <div>{count}</div>;
}

Using useState keeps everything predictable and working correctly.

Should You Use It?

Yes! Always wrap your app in Strict Mode. It helps catch bugs early and keeps your code clean. You won't even notice it's there, except when it helpfully points out problems.

useState Hook

What is useState?

useState lets your component "remember" information between renders. Without it, variables reset every time your component re-renders.

Basic Syntax
import { useState } from 'react';

function Counter() {
  // Declare a state variable
  const [count, setCount] = useState(0);
  //      ↑ current value
  //             ↑ function to update it
  //                           ↑ initial value
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}

Why Do We Need useState?

❌ Without useState

function Counter() {
  let count = 0; // Resets to 0 every render!
  
  return (
    <button onClick={() => count++}>
      {count}
    </button>
  );
}

This won't work - the count resets every time.

✅ With useState

function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

React remembers the count between renders!

Common useState Patterns:

Different Types of State
// Numbers
const [age, setAge] = useState(25);
setAge(26);

// Strings
const [name, setName] = useState('Alice');
setName('Bob');

// Booleans
const [isOpen, setIsOpen] = useState(false);
setIsOpen(true);

// Arrays
const [items, setItems] = useState([]);
setItems(['apple', 'banana']);

// Objects
const [user, setUser] = useState({ name: 'Alice', age: 25 });
setUser({ name: 'Bob', age: 30 });

// Update based on previous value (safer for async updates)
setCount(prevCount => prevCount + 1);

Key Points:

  • useState returns an array with two items: [value, updater function]
  • Calling the updater function causes a re-render with the new value
  • State updates are asynchronous - they don't happen immediately
  • Never modify state directly - always use the updater function

useEffect Hook

What is useEffect?

useEffect lets you run code after your component renders. It's perfect for things like fetching data, setting up timers, or connecting to external services.

1. Mount

Component appears on screen

useEffect runs

→

2. Update

Data changes

useEffect runs again

→

3. Unmount

Component removed

Cleanup runs

Basic Syntax
import { useEffect } from 'react';

useEffect(() => {
  // This code runs after the component renders
  console.log('Component rendered!');
  
  // Optional: return a cleanup function
  return () => {
    console.log('Cleaning up...');
  };
}, [dependencies]); // Controls when effect runs

The Dependency Array Explained:

No Dependencies

useEffect(() => {
  console.log('Runs after every render');
});
// No second argument

Runs after every single render. Rarely needed.

Empty Array []

useEffect(() => {
  console.log('Runs once on mount');
}, []);
// Empty array

Runs only once when component first appears.

With Dependencies

useEffect(() => {
  console.log('Count changed!');
}, [count]);
// Runs when count changes

Runs when count changes.

Example: Fetching Data 

import { useState, useEffect } from 'react';

function UserList() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    // Fetch runs once on mount
    fetch('https://jsonplaceholder.typicode.com/users')
      .then(res => res.json())
      .then(data => setUsers(data));
  }, []);

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

useEffect runs once when the component mounts and fetches user data.

Cleanup Example:

Some effects like event listeners or timers should be cleaned up to avoid memory leaks. 

useEffect(() => {
  const interval = setInterval(() => {
    console.log('Tick');
  }, 1000);

  // Cleanup function runs when component unmounts
  return () => clearInterval(interval);
}, []);

Key Points:

  • useEffect runs after every render by default
  • Use [] to run only once (on mount)
  • Add dependencies to control when it re-runs
  • Return a cleanup function for unmount logic
